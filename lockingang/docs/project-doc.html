Executive Summary

lockingang is a local-first desktop application that solves the problem of passive, inefficient study for students and self-learners. Its core — the Learning State Engine — continuously models what a student knows, how fast they are forgetting it, and what single action they should take next to maximise long-term retention.

The Core Problem

Students spend hours re-reading notes passively without knowing which topics they are actually forgetting. They cannot answer: What am I genuinely weak at? Am I improving or regressing? What should I focus on right now? lockingang solves this with a live, per-topic knowledge score that decays in real time, driving personalised quiz selection, intelligent scheduling, and AI-generated guidance.

Mathematical Analogy

Knowledge is modelled as a vector field V where each concept is a point. Without active review, entropy causes the magnitude of these vectors (mastery) to shrink toward zero. The engine acts as an External Impulse Force that calculates the necessary work (active recall) required to counteract decay and maintain high-energy mastery.

02

Design Principles

1 — Constraint over Freedom

The Rule of 3 prevents choice paralysis. The active task queue has a hard limit of 3 items. Limitations are features.

2 — Active over Passive

Every interaction drives active recall. Re-reading is replaced by quizzes, and progress requires demonstrated understanding.

3 — Visual over Textual

The Fog of War overlay, forgetting forecasts, and colour-coded nodes make memory decay tangible and intuitive rather than abstract.

4 — Compassion over Shame

Bridge nodes and grandparent resets treat failure as structural data points to be solved — not personal flaws. The system adapts to the student.

5 — Explainable over Opaque

Every recommendation comes with reasoning. The student always understands why the system is suggesting a particular action.

6 — Adaptive over Static

The system continuously evolves its model of the student. Long gaps, bursts of activity, and shifting priorities are all handled gracefully.

Dashboard

The first screen a returning student sees. Displays the urgency queue (top nodes most in need of review), a mastery distribution overview, trend analysis (improving / stable / regressing), 7-day forgetting forecast, and integrated chatbot access. Also hosts the Focus Tunnel queue for task management.

Knowledge Tree

The heart of lockingang. An interactive, zoomable graph where each node represents a concept and each edge a relationship. Nodes are colour-coded by urgency:

Blue — Mastered

Live score > 0.7. Node is strong.

Yellow — Fading

Score 0.3–0.7. Review soon.

Red — Critical

Score < 0.3. Quiz due.

Edge types: requires (prerequisite), is_a_type_of (taxonomy), related_to (association), bridges_to (AI-generated transition).

Visual Overlays: Fog of War (nodes < 0.3 are dimmed), Forgetting Forecast (7-day heatmap), Urgency Pulse (animated glow on critical nodes).

Focus Tunnel

A distraction-free task execution environment. The active queue is hard-limited to 3 tasks (Rule of 3). Users add tasks by name and duration. Clicking Start enters Zen Mode — a full-screen countdown timer that counts down from the user-set duration. On completion, the student is prompted to connect the session to a Knowledge Tree node.

Chatbot

An agentic AI assistant deeply integrated with the student's knowledge graph. Capabilities include RAG-powered Q&A (grounded in the student's own notes via Pinecone), graph manipulation via natural language, calendar and Todoist actions, and multi-day exam study plan generation.

Templates

A library of pre-built learning roadmaps. Students can browse, import, and clone community-shared trees. Importing clones the full structure (nodes, edges, notes, pre-made quizzes) into the student's own graph. Students can also export and share their own trees as templates.

Calendar & Schedule

A unified weekly view combining Google Calendar events with lockingang's auto-scheduled review sessions. Events are colour-coded by node urgency. Review blocks can be dragged to reschedule within a 1-week window. Clicking a review block opens the relevant node's quiz panel.

06

Forgetting Curve Engine

Every node has a real-time competence score that decays continuously based on how long ago it was last reviewed and the node's individual decay rate.

live_score = stored_score × (0.5 ^ (days_elapsed / decay_rate))
ParameterDescription
stored_scoreMastery after the last quiz: 1.0 (correct) or 0.2 (incorrect)
days_elapsedFractional days since last_reviewed timestamp
decay_ratePer-node half-life in days (default 5). Higher = slower decay.
live_scoreCurrent effective mastery, clamped to [0.0, 1.0]

A background decay cycle runs hourly. When a node transitions from blue → yellow, a review session is auto-scheduled. When it crosses yellow → red (or is within 24 hours of the scheduled quiz), it turns red and appears at the top of the urgency queue. Completing the quiz resets the decay clock.

07

Quiz Engine

Every quiz interaction updates the student's competence scores and feeds the adaptive logic engine. The quiz flow is:

1
Kahoot Search — Query the Kahoot API with the node title as the search term. Retrieve top 5 candidate quizzes.
2
Kahoot Evaluation — GPT-4o scores each candidate for relevance (0–1) and difficulty. Accept if relevance ≥ 0.7, adapt/reword questions as needed.
3
RAG Fallback — If no Kahoot scores ≥ 0.7, embed the node title, retrieve top 6 chunks from the node's own FAISS index, and generate 5–10 MCQs via GPT-4o grounded strictly in the student's material.
4
Synthesis Questions (Diamond Problem) — If the node has 2+ parents, append 1–2 synthesis questions requiring integrated understanding from both parent concepts.
5
Scoring — Correct answer → mastery = 1.0. Incorrect → mastery = 0.2. Decay clock resets on every attempt.
08

Adaptive Logic Engine

The adaptive engine identifies when a student is structurally stuck and intervenes automatically — without blame.

Wall Detection

If a student fails a child node's quiz 3+ consecutive times while the parent node shows mastery (> 0.7), the system flags it as The Wall. This indicates a gap in foundational understanding between the parent and child concepts.

Bridge Node Generation

Upon Wall detection, the system automatically generates a new bridge node between the parent and the difficult child. GPT-4o writes a 300–500 word introductory markdown note focused on the specific gap. The bridge node is inserted into the graph as: parent → bridges_to → bridge → bridges_to → child.

Grandparent Reset

If the wall persists (the bridge node already exists and the student still fails), the system escalates by resetting the grandparent node's mastery to 0.0, forcing a review of the deeper fundamentals driving the comprehension gap.

10

Auto-Scheduler

Because lockingang tracks per-topic mastery, decay rates, and quiz history, it can schedule study sessions with precision rather than guesswork.

Forgetting Curve Optimiser

  • Projects when each node will fall below the critical threshold (0.3)
  • Generates optimal review slots before the threshold is crossed
  • Auto-inserts review blocks into Google Calendar (colour-coded blue/yellow/red)
  • Sessions recalculated daily by the background decay cycle

Exam Preparation Mode

  • Scope Detection: Identifies all nodes relevant to the exam topic
  • Gap Analysis: Highlights nodes below target competence (0.7)
  • Plan Generation: Creates a multi-day plan front-loading weak topics
  • Calendar Injection: Approved sessions scheduled with direct node links

Ideal User Flow

1
First-Time Setup — Import a template or upload course materials. AI generates the knowledge tree. Connect Google Calendar and Todoist.
2
Daily Study Session — Check Dashboard briefing → move top urgency items to Focus Tunnel → enter Zen Mode → complete session. Stray thoughts captured via Brain Dump (Cmd+J). Decay clock recalculates overnight.
3
Exam Preparation — Tell the Chatbot your exam date. It proposes an N-day study plan and injects sessions into Google Calendar for confirmation.
4
Ongoing Adaptation — Wall Detection automatically inserts bridge nodes when a student repeatedly fails a concept. Grandparent resets escalate to deeper fundamentals when needed.
